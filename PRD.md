# PRD: 생산증권

## 1. 제품 한 줄 설명

생산증권은 사용자의 감정, 일기, 가용 시간, 오늘의 할 일, KOSPI 시장 맥락을 함께 보여주고 GitHub Copilot 기반 코칭으로 다음 집중 행동을 정리해 주는 개인 생산성 웹앱입니다.

## 2. 문제 정의

개인은 하루 계획을 세울 때 단순히 할 일 목록만 보는 것이 아니라 컨디션, 감정, 남은 시간, 외부 변수까지 함께 고려해야 합니다. 특히 투자자나 시장 흐름에 민감한 사용자는 KOSPI 변화가 기분과 집중도에 영향을 줄 수 있습니다. 생산증권은 이 정보를 한 화면에 모아 사용자가 오늘 할 일을 더 현실적으로 조정하도록 돕습니다.

## 3. 목표 사용자

- 오늘의 작업 우선순위를 빠르게 정하고 싶은 개인 사용자
- 일기와 기분 점수를 기록하며 생산성 패턴을 보고 싶은 사용자
- KOSPI 시장 흐름이 자신의 기분과 집중 상태에 영향을 준다고 느끼는 사용자
- GitHub Copilot 기반의 짧고 실행 가능한 코칭을 받고 싶은 사용자

## 4. 핵심 가치

- 감정 점수와 일기를 생산성 계획의 입력으로 사용합니다.
- 최근 기분 흐름을 홈 화면 차트로 보여줍니다.
- KOSPI 데이터는 목 데이터가 아니라 백엔드에서 pykrx로 조회한 실제 데이터만 사용합니다.
- 실제 KOSPI 데이터를 가져오지 못하면 가짜 차트를 보여주지 않고 명확한 데이터 없음 상태를 표시합니다.
- GitHub Copilot SDK를 사용해 오늘의 투두, 가용 시간, 감정, 일기 맥락을 기반으로 생산성 코칭을 제공합니다.

## 5. MVP 범위

### 5.1 홈

- 서비스명 `생산증권`을 표시합니다.
- 최근 7일 기분 점수 데모 데이터를 기본 제공해 배포 환경에서도 감정 차트를 바로 확인할 수 있습니다.
- KOSPI 차트 영역은 `/api/markets/kospi` 응답을 기반으로 표시합니다.
- KOSPI 데이터가 없거나 pykrx 조회가 실패하면 `현재 코스피 데이터를 불러올 수 없습니다.` 상태를 표시합니다.

### 5.2 일기와 기분 점수

- 사용자는 오늘의 일기와 1부터 10 사이의 기분 점수를 저장할 수 있습니다.
- 저장된 일기는 뉴스 피드처럼 보관함에 표시됩니다.
- 기분 점수 차트는 일기 보관함의 날짜별 평균 기분 점수를 사용합니다.
- MVP 데모를 위해 서버 시작 시 최근 기분 점수 목 데이터를 제공합니다.

### 5.3 KOSPI 시장 데이터

- 백엔드는 `pykrx`를 사용해 KOSPI 지수 티커 `1001`의 최근 데이터를 조회합니다.
- 프런트엔드는 KOSPI 목 데이터를 포함하지 않습니다.
- 백엔드는 KOSPI 조회 실패 시 Naver 등 다른 시세 소스로 대체하지 않습니다.
- 데이터가 없으면 API는 명확한 오류를 반환하고, 프런트엔드는 데이터 없음 상태를 표시합니다.

### 5.4 투두와 가용 시간

- 사용자는 오늘의 할 일을 추가, 완료, 삭제할 수 있습니다.
- 사용자는 가용 시간대를 입력해 전체 가용 시간을 계산할 수 있습니다.
- 오늘의 투두리스트와 다음날 추천 투두리스트는 브라우저 로컬 저장소에 보관합니다.

### 5.5 GitHub Copilot 기반 코치

- 사용자는 Copilot 코치 탭에서 자연어 요청을 보낼 수 있습니다.
- 백엔드는 GitHub Copilot SDK를 사용해 사용자의 투두, 가용 시간, 일기, 기분 점수, KOSPI 맥락을 바탕으로 답변합니다.
- Copilot 토큰이 없거나 모델 호출에 실패하면 사용자에게 명확한 오류 메시지를 반환합니다.

## 6. 주요 API

- `GET /health`: 서비스 상태 확인
- `GET /`: Vue 정적 프런트엔드 제공
- `POST /api/user-profile/summary`: 사용자 이름, 가용 시간, KOSPI 투자 여부 요약
- `POST /api/diaries/today`: 오늘의 일기와 기분 점수 저장
- `GET /api/diaries/today`: 오늘의 일기 조회
- `GET /api/diaries`: 일기 보관함 조회
- `GET /api/markets/kospi`: pykrx 기반 KOSPI 최근 지수 데이터 조회
- `POST /api/productivity/plan`: 오늘의 집중 계획 생성
- `POST /api/productivity/next-day-todos`: 다음날 투두리스트 생성
- `POST /api/productivity/coach`: GitHub Copilot SDK 기반 생산성 코칭
- `/copilotkit`: CopilotKit 액션 엔드포인트

## 7. 성공 기준

- 공개 GitHub 저장소 루트에 이 `PRD.md` 파일이 존재합니다.
- Azure 배포 URL에서 `생산증권` 홈 화면이 열립니다.
- 배포 환경에서 기분 점수 차트가 기본 데모 데이터로 표시됩니다.
- KOSPI는 프런트엔드 목 데이터가 아니라 pykrx API 결과만 사용합니다.
- pykrx가 데이터를 반환하지 않는 경우 가짜 데이터 대신 데이터 없음 상태가 표시됩니다.
- GitHub Copilot SDK 기반 코치 기능이 별도 탭에서 확인됩니다.

## 8. 배포 정보

- 배포 대상: Azure App Service
- 현재 배포 URL: https://vibe-copilot-mac-06201119.azurewebsites.net/
- 백엔드: Python FastAPI
- 프런트엔드: Vue 3 CDN 기반 정적 화면
- 주요 런타임 명령: `python -m uvicorn main:app --host 0.0.0.0 --port 8000`
