# 기능명세서

## 1. 서비스 개요

`FocusFlow`는 개인 생산성 향상을 돕는 웹앱입니다. 사용자는 오늘 처리할 작업, 현재 에너지 상태, 사용 가능한 시간을 입력하고, Copilot 기반 추천을 통해 다음 집중 블록과 실행 순서를 정합니다.

## 2. 필수 조건

- 웹 앱으로 제공한다.
- Copilot SDK를 사용한다.
- Azure 플랫폼에 배포한다.
- 프런트엔드와 백엔드 작업 공간을 분리한다.

## 3. 담당 영역

| 영역 | 담당 | 주요 경로 |
|------|------|----------|
| 프런트엔드 | Vue/JavaScript 담당 | `frontend/` |
| 백엔드 | Python/FastAPI 담당 | `backend/` |
| API 계약 | 백엔드 주도, 프런트와 공유 | `API.md` |
| 배포 | 백엔드 주도 | `DEPLOY.md`, `startup.sh` |

## 4. MVP 기능

### 4.1 사용자 정보 입력

- 사용자는 계획 생성을 위해 기본 사용자 정보를 입력해야 한다.
- 사용자는 하루 근무 시간을 입력해야 한다.
- 사용자는 하루 수면 시간을 입력해야 한다.
- 사용자는 근무 시간과 수면 시간을 제외한 가용시간을 반드시 입력해야 한다.
- 가용시간은 집중 계획 생성의 핵심 입력값으로 사용된다.
- 가용시간이 입력되지 않으면 계획 생성 요청을 진행할 수 없다.

### 4.2 작업 관리

- 사용자는 작업 제목을 입력해 작업을 추가할 수 있다.
- 사용자는 작업 우선순위를 지정할 수 있다.
- 지원 우선순위는 `high`, `normal`, `medium`, `low`이다.
- 사용자는 작업을 완료 처리하거나 목록에서 제거할 수 있다.
- 작업 목록은 Copilot 계획 생성 요청의 입력값으로 사용된다.

### 4.3 오늘의 투두리스트 입력

- 사용자는 오늘 수행해야 할 투두리스트를 입력해야 한다.
- 투두리스트는 여러 개의 할 일 항목으로 구성된다.
- 각 투두 항목은 제목을 필수로 가진다.
- 각 투두 항목은 우선순위와 완료 여부를 선택적으로 가진다.
- 오늘의 투두리스트가 비어 있으면 집중 계획 생성 전에 투두 입력을 안내한다.
- 오늘의 투두리스트는 Copilot 계획 생성과 생산성 코치 답변의 핵심 입력값으로 사용된다.

### 4.4 오늘의 투두리스트 보관함

- 사용자는 날짜별 오늘의 투두리스트를 보관할 수 있다.
- 각 보관 항목은 날짜, 투두 목록, 완료 상태, 총 예상 소요 시간을 포함한다.
- 사용자는 과거 날짜의 오늘의 투두리스트를 조회할 수 있다.
- 사용자는 과거 투두리스트에서 완료하지 못한 항목을 다음 계획 생성에 참고할 수 있다.
- MVP에서는 브라우저 로컬 저장소를 사용해 오늘의 투두리스트 보관함을 제공한다.
- 계정 기반 서버 보관은 향후 확장 범위로 둔다.

### 4.5 컨디션 입력

- 사용자는 현재 에너지 상태를 선택할 수 있다.
- 지원 에너지 값은 `low`, `medium`, `high`이다.
- 사용자는 오늘 사용할 수 있는 집중 시간을 분 단위로 입력할 수 있다.
- 입력된 컨디션은 계획 생성 로직에 반영된다.

### 4.6 일기 및 기분 점수 입력

- 사용자는 오늘의 일기를 입력할 수 있다.
- 사용자는 오늘의 기분 점수를 입력해야 한다.
- 기분 점수는 숫자 값으로 입력한다.
- 기분 점수는 사용자의 현재 상태를 판단하는 핵심 입력값으로 사용된다.
- 일기 내용은 Copilot 생산성 코치가 사용자의 감정, 부담, 집중 방해 요인을 파악하는 참고 정보로 사용된다.
- 기분 점수가 입력되지 않으면 Copilot 코치 요청을 진행할 수 없다.

### 4.7 일기 보관함

- 사용자는 작성한 일기를 날짜별로 보관할 수 있다.
- 사용자는 보관함에서 과거 일기 목록을 확인할 수 있다.
- 각 일기 항목은 작성일, 일기 내용, 기분 점수, 가용시간 정보를 포함한다.
- 사용자는 보관된 일기를 선택해 상세 내용을 확인할 수 있다.
- MVP에서는 브라우저 로컬 저장소를 사용해 일기 보관함을 제공한다.
- 서버 저장소 기반 영구 보관은 향후 확장 범위로 둔다.

### 4.8 기분 점수 차트

- 사용자는 입력한 기분 점수를 기반으로 차트를 확인할 수 있다.
- 홈 화면에는 감정 차트가 표시되어야 한다.
- 차트는 일기 보관함의 날짜별 기분 점수를 사용한다.
- 차트는 최근 기분 흐름을 시각적으로 보여준다.
- 차트에는 최소한 날짜와 기분 점수가 표시되어야 한다.
- 데이터가 부족한 경우 차트 대신 입력 안내 상태를 표시한다.

### 4.9 집중 계획 생성

- 사용자는 `Ask Copilot` 버튼을 눌러 집중 계획을 요청할 수 있다.
- 백엔드는 작업, 에너지 상태, 근무/수면 제외 가용시간을 기준으로 집중 블록을 생성한다.
- 응답에는 다음 항목이 포함된다.
  - 요약 문구
  - 다음 행동
  - 집중 블록 목록
  - Copilot 기반 생성 안내 문구

### 4.10 다음날 투두리스트 생성

- 사용자는 오늘의 일기, 오늘의 기분 점수, 근무/수면 제외 가용시간을 기반으로 다음날 투두리스트를 생성할 수 있다.
- 백엔드는 일기 내용에서 사용자의 부담, 미완료 작업, 에너지 상태를 참고한다.
- 백엔드는 기분 점수가 낮을수록 더 작고 현실적인 투두를 추천한다.
- 백엔드는 가용시간을 초과하지 않도록 다음날 투두리스트를 구성한다.
- 생성 결과에는 할 일 제목, 우선순위, 예상 소요 시간이 포함되어야 한다.
- 생성된 다음날 투두리스트는 사용자가 오늘의 투두리스트로 가져올 수 있어야 한다.

### 4.11 다음날 투두리스트 보관함

- 사용자는 생성된 다음날 투두리스트를 날짜별로 보관할 수 있다.
- 각 보관 항목은 생성일, 대상 날짜, 추천 투두 목록, 총 예상 소요 시간, 추천 이유를 포함한다.
- 사용자는 과거에 생성된 다음날 투두리스트를 조회할 수 있다.
- 사용자는 보관된 다음날 투두리스트를 오늘의 투두리스트로 가져올 수 있다.
- MVP에서는 브라우저 로컬 저장소를 사용해 다음날 투두리스트 보관함을 제공한다.
- 계정 기반 서버 보관은 향후 확장 범위로 둔다.

### 4.12 Copilot 생산성 코치

- 사용자는 생산성 코치 API에 자연어 요청을 보낼 수 있다.
- 백엔드는 GitHub Copilot SDK를 통해 사용자의 작업 상태, 가용시간, 일기, 기분 점수를 바탕으로 코칭 답변을 생성한다.
- 토큰이 설정되지 않은 경우 명확한 오류 메시지를 반환한다.

## 5. 화면 구성

### 5.1 메인 화면

- 서비스명과 현재 SDK 상태를 표시한다.
- 사용자 정보 입력 영역을 제공한다.
- 근무 시간 입력 컨트롤을 제공한다.
- 수면 시간 입력 컨트롤을 제공한다.
- 근무 시간과 수면 시간을 제외한 가용시간 입력 컨트롤을 제공한다.
- 에너지 선택 컨트롤을 제공한다.
- 오늘의 일기 입력 영역을 제공한다.
- 오늘의 기분 점수 입력 컨트롤을 제공한다.
- 일기 보관함 영역을 제공한다.
- 홈 화면 감정 차트 영역을 제공한다.
- 사용 가능한 시간 입력 컨트롤을 제공한다.
- 오늘의 투두리스트 입력 영역을 제공한다.
- 오늘의 투두리스트 보관함 영역을 제공한다.
- 작업 추가 폼을 제공한다.
- 작업 목록 패널을 제공한다.
- Copilot 계획 결과 패널을 제공한다.
- 다음날 투두리스트 생성 결과 영역을 제공한다.
- 다음날 투두리스트 보관함 영역을 제공한다.

### 5.2 계획 결과 영역

- 아직 계획이 없을 때 안내 문구를 표시한다.
- 계획 생성 중 로딩 상태를 표시한다.
- 계획 생성 실패 시 오류 상태를 표시한다.
- 계획 성공 시 다음 행동과 집중 블록을 표시한다.

### 5.3 일기 보관함 화면

- 날짜별 일기 목록을 표시한다.
- 각 항목에는 날짜, 기분 점수, 일기 일부 내용을 표시한다.
- 사용자가 항목을 선택하면 상세 일기 내용을 표시한다.
- 보관된 기분 점수 데이터가 있으면 차트에 반영한다.

### 5.4 홈 화면 감정 차트 영역

- 홈 화면에서 최근 감정 흐름 차트를 표시한다.
- 차트 데이터는 일기 보관함의 날짜별 `mood_score`를 사용한다.
- 차트는 최근 항목을 시간 순서대로 표시한다.
- 차트에는 각 점수의 날짜와 점수를 확인할 수 있어야 한다.
- 데이터가 없거나 부족할 경우 일기와 기분 점수 입력을 안내한다.

### 5.5 다음날 투두리스트 결과 영역

- 생성된 다음날 투두리스트를 표시한다.
- 각 항목에는 제목, 우선순위, 예상 소요 시간을 표시한다.
- 총 예상 시간이 가용시간 안에 들어오는지 표시한다.
- 사용자가 결과를 오늘의 투두리스트로 가져올 수 있는 액션을 제공한다.

### 5.6 투두리스트 보관함 화면

- 오늘의 투두리스트 보관함과 다음날 투두리스트 보관함을 구분해 표시한다.
- 각 보관함은 날짜별 목록을 제공한다.
- 오늘의 투두리스트 보관 항목에는 완료율과 미완료 항목 수를 표시한다.
- 다음날 투두리스트 보관 항목에는 총 예상 소요 시간과 대상 날짜를 표시한다.
- 사용자는 보관된 투두리스트를 선택해 상세 내용을 확인할 수 있다.

## 6. 백엔드 API

자세한 API 계약은 `API.md`를 기준으로 한다.

### 6.1 헬스체크

```http
GET /health
```

### 6.2 집중 계획 생성

```http
POST /api/productivity/plan
```

주요 입력값:

- `tasks`
- `today_todos`
- `energy`
- `available_minutes`
- `work_minutes`
- `sleep_minutes`
- `free_minutes`
- `diary`
- `mood_score`

### 6.3 Copilot 생산성 코치

```http
POST /api/productivity/coach
```

주요 입력값:

- `request`
- `tasks`
- `today_todos`
- `energy`
- `mood`
- `focus_sessions`
- `work_minutes`
- `sleep_minutes`
- `free_minutes`
- `diary`
- `mood_score`

### 6.4 다음날 투두리스트 생성

```http
POST /api/productivity/next-day-todos
```

주요 입력값:

- `diary`
- `mood_score`
- `free_minutes`
- `today_todos`
- `unfinished_tasks`

주요 응답값:

- `tomorrow_todos`
- `total_estimated_minutes`
- `reason`

### 6.5 일기 보관함

MVP에서는 프런트엔드 로컬 저장소를 사용한다. 서버 API가 필요한 경우 아래 계약으로 확장한다.

```http
GET /api/diaries
POST /api/diaries
GET /api/diaries/{date}
```

### 6.6 투두리스트 보관함

MVP에서는 프런트엔드 로컬 저장소를 사용한다. 서버 API가 필요한 경우 아래 계약으로 확장한다.

```http
GET /api/todo-archives/today
POST /api/todo-archives/today
GET /api/todo-archives/tomorrow
POST /api/todo-archives/tomorrow
GET /api/todo-archives/{date}
```

### 6.7 기분 점수 차트 데이터

MVP에서는 일기 보관함의 로컬 데이터를 사용한다. 서버 API가 필요한 경우 아래 계약으로 확장한다.

```http
GET /api/mood-scores
```

### 6.8 CopilotKit 엔드포인트

```http
/copilotkit
```

프런트엔드에서 CopilotKit 연동을 확장할 때 사용한다.

## 7. 데이터 모델

### 7.1 User Availability

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `work_minutes` | number | 예 | 하루 근무 시간 |
| `sleep_minutes` | number | 예 | 하루 수면 시간 |
| `free_minutes` | number | 예 | 근무 시간과 수면 시간을 제외한 가용시간 |
| `energy` | string 또는 number | 예 | 현재 에너지 상태 |
| `mood` | string | 아니오 | 현재 컨디션 또는 감정 상태 |
| `mood_score` | number | 예 | 오늘의 기분 점수 |
| `diary` | string | 아니오 | 오늘의 일기 내용 |

### 7.2 Task

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `title` | string | 예 | 작업 제목 |
| `priority` | string | 아니오 | 작업 우선순위 |
| `done` | boolean | 아니오 | 완료 여부 |

### 7.3 Today Todo

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `title` | string | 예 | 오늘 수행할 할 일 제목 |
| `priority` | string | 아니오 | 오늘 할 일의 우선순위 |
| `done` | boolean | 아니오 | 완료 여부 |
| `estimated_minutes` | number | 아니오 | 예상 소요 시간 |

### 7.4 Today Todo Archive Entry

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `date` | string | 예 | 투두리스트 날짜 |
| `today_todos` | Today Todo[] | 예 | 해당 날짜의 오늘 투두리스트 |
| `completed_count` | number | 아니오 | 완료 항목 수 |
| `unfinished_count` | number | 아니오 | 미완료 항목 수 |

## 8. 비기능 요구사항

- 로컬 개발 환경에서 프런트엔드와 백엔드 충돌을 최소화한다.
- API 오류는 프런트엔드에서 표시 가능한 명확한 메시지로 반환한다.
- 환경 변수와 API 키는 코드에 포함하지 않는다.
- Azure App Service 배포를 지원한다.
- 백엔드는 Vue 개발 서버 origin을 CORS로 허용한다.
